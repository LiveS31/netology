важно - первичный ключ primary key
  - столбец или набор столбцов, которые гарантируют уникальным образом индифицирют
   строку в рамках все таблици
   (если просто - ключ который гарантирует уникальность поиска набора)
  - Foreign key
    внешний ключ позволяющий установить связи между таблицами
    внешний ключ устанавливается для столбцов из зависимой (подчиненной) таблицы
    и указывает на один из столбцов из главной таблицы
    Как правило, внешний ключ указывает на первичный ключ из главной таблицы
  - Нормальная форма - требования предъявляемое к структуре таблиц в теории реляционных
    баз данных для устранения из базы избыточных функциональных зависимостей межды
    атрибутами
  - Цель нормализации - исключить избыточное дублирование данных, которое является причиной     аномалий, возникающих при добовлении, редактировании и удалении кортежей.
Первая Нормальная форма
 -переменная отношения находится В ПЕРВОЙ НОРМАЛЬНОЙ ФОРМЕ (1ФН)
 в таблице не рекомендуется более одного значения в одной ячейки
Вторая нормальная форма -
 - подрузумевает соблюдение всех требований предъидущей и каждый неключевой атрибут неприводимо (функционально полно) зависит от ее потенциального ключа
Третья нормальная форма
 - подрузумевает соблюдение всех требовании предъидущих форм
   отсутствуют функциональный зависимости неключевых атрибутов от ключевых
